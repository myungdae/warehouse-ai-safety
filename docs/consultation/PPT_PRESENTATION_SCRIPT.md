## 슬라이드 2: 문제 정의

### 화면
```
현재 물류센터의 문제

📊 통계:
- 연간 충돌 사고: 10-15건
- 사고당 평균 비용: 3,000만원
- 연간 총 손실: 3-5억원

⚠️ 주요 원인:
- 블라인드 코너
- 교차로 충돌
- 지게차-보행자 접촉
- 보행자 구역 과속

[사고 현장 사진 4장]
```

### 발표 (1분)
"현재 물류센터에서 가장 심각한 문제는 지게차 관련 사고입니다.

통계를 보시면, 연간 평균 10-15건의 충돌 사고가 발생하고 있습니다.  
사고 한 건당 평균 3,000만원의 비용이 발생하죠.  
여기에는 차량 수리비, 산재 보상, 생산 중단 손실이 포함됩니다.

특히 심각한 것은 보행자와의 충돌입니다.  
인명 피해가 발생하면 보상금은 물론, 법적 책임, 기업 이미지 실추까지 이어집니다.

**사고가 발생하는 주요 원인은 4가지입니다:**
1. 블라인드 코너에서의 갑작스러운 조우
2. 교차로에서 양쪽 지게차의 충돌
3. 지게차와 보행자의 접촉
4. 보행자 구역에서의 과속

이 문제들을 어떻게 해결할 수 있을까요?"

## 슬라이드 3: 기존 솔루션의 한계

### 화면
```
기존 시스템의 한계

❌ CCTV 녹화
- 사후 대응만 가능
- 사고 예방 불가
- 24시간 수동 감시 필요

❌ 단순 위치 추적
- 위험 판단 불가
- 개입 방법 없음
- 데이터만 수집

❌ 자율 주행 지게차
- 비용: 대당 2-3억원
- 전면 교체 필요
- 책임 소재 불명확
- ROI: 5-7년
```

### 발표 (1분)
"많은 물류센터에서 CCTV를 설치하고 있습니다.  
하지만 CCTV는 사후 녹화만 가능할 뿐, 사고를 예방할 수 없습니다.  
관제 요원이 24시간 화면을 주시해야 하는데, 인간의 집중력에는 한계가 있죠.

일부 기업은 RTLS 같은 위치 추적 시스템을 도입했습니다.  
지게차가 어디에 있는지는 알 수 있지만, 그게 위험한지 판단할 수 없습니다.  
그리고 위험을 알아도 개입할 방법이 없습니다.

자율 주행 지게차는 어떨까요?  
이론적으로는 완벽하지만, 현실은 다릅니다.  
대당 2-3억원이라는 엄청난 비용, 전체 장비를 교체해야 하는 부담,  
그리고 가장 중요한 것 - 사고가 발생하면 누구 책임인가요?

ROI를 회수하는 데 5-7년이 걸립니다.  
대부분의 기업이 선택할 수 없는 옵션입니다.

**우리는 다른 접근 방식이 필요합니다.**"

## 슬라이드 4: 우리의 솔루션

### 화면
```
Warehouse Digital Twin
실시간 교통 관제 시스템

[Digital Twin 대시보드 스크린샷]

핵심 기능:
✅ 실시간 위치 추적 (±30cm 정확도)
✅ AI 충돌 위험 예측 (0.2초 이내)
✅ 음성 명령 자동 발령
✅ 모든 이벤트 기록

투자: 2.3억원
회수: 6개월
```

### 발표 (1분 30초)
"우리의 솔루션은 **Warehouse Digital Twin**입니다.

화면에 보이시는 것이 실제 작동하는 시스템입니다.  
왼쪽에는 물류센터의 실시간 지도가 있고,  
파란색 사각형들이 지게차, 초록색 점들이 보행자입니다.

**시스템은 4가지 핵심 기능을 제공합니다:**

첫째, UWB 센서로 ±30cm 정확도로 위치를 추적합니다.  
둘째, AI가 0.2초 이내에 충돌 위험을 예측합니다.  
셋째, 위험 감지 시 즉시 음성 명령을 발령합니다.  
넷째, 모든 판단과 행동을 기록합니다.

**그리고 가장 중요한 차이점:**

이것은 자율 제어가 아닙니다.  
시스템은 조언자(Advisor)입니다.  
최종 결정은 운전자(Human)가 내립니다.

초기 투자는 2.3억원입니다.  
자율 주행 지게차 1대 값입니다.  
하지만 회수 기간은 단 6개월입니다.

어떻게 가능할까요? 지금 실제 데모를 보여드리겠습니다."

## 슬라이드 5: 데모 영상 재생

### 화면
```
[영상 재생: 1분]

시나리오 1: 충돌 위험 (20초)
시나리오 2: 보행자 근접 (20초)
시나리오 3: 과속 감지 (20초)
```

### 발표 (데모 중 내레이션)

**시나리오 1 시작 (0:00-0:20)**  
"첫 번째 시나리오를 보시겠습니다.  
빨간 버튼을 클릭하면... (클릭)

들으셨나요? '시나리오 1. 충돌 위험 감지'라는 음성이 나왔습니다.

두 대의 지게차가 교차로로 접근하고 있습니다.  
F 대시 07과 F 대시 12입니다.

시스템이 충돌 위험을 감지했습니다! (경고음)  
즉시 음성 명령이 발령됩니다:

'F 대시 07! 정지! 교차로 확인!'  
'F 대시 12! 정지! 교차로 확인!'

오른쪽 패널을 보시면 음성 명령 로그가 실시간으로 기록되고,  
충돌 방지 카운터가 1 증가했습니다.

**이것이 실제로 작동하는 모습입니다.**"

**시나리오 2 시작 (0:20-0:40)**  
"두 번째는 보행자 근접 시나리오입니다. (클릭)

F 대시 03 지게차가 보행자에게 빠르게 접근하고 있습니다.  
거리가 3미터 이내로 가까워지자...

시스템이 즉시 반응합니다! (경고음)  
'F 대시 03! 긴급정지! 보행자!'

위험 레벨이 CRITICAL로 표시되고,  
지게차가 즉시 정지합니다.

**사고를 미연에 방지했습니다.**"

**시나리오 3 시작 (0:40-1:00)**  
"마지막은 과속 감지입니다. (클릭)

F 대시 15 지게차가 보행자 구역(초록색 영역)에 진입합니다.  
속도가 15km/h로 제한 속도 5km/h를 초과하고 있습니다.

시스템이 감지하고 (경고음)  
'F 대시 15! 감속! 보행자 구역!'

자동으로 속도가 5km/h로 제한됩니다.

**3가지 시나리오 모두 성공적으로 처리되었습니다.**  
충돌 방지 카운터를 보시면 3건이 기록되어 있습니다.

이것이 단 2주간의 테스트에서 얻을 수 있는 결과입니다.  
1년이면 얼마나 많은 사고를 방지할 수 있을까요?"

## 슬라이드 6: 핵심 기술

### 화면
```
핵심 기술: 온톨로지 기반 지식 시스템

센서 → 온톨로지 → 추론 → 정책 → 행동
(Raw)   (Meaning)  (Reason) (Policy) (Action)

SWRL 추론 규칙 예시:
IF 지게차 2대가 교차로 접근
AND 속도 > 8km/h
THEN 충돌 위험 HIGH
     음성 명령: "정지!"

[Palantir 스타일 다이어그램]
```

### 발표 (1분)
"이 시스템의 핵심은 온톨로지 기반 지식 모델링입니다.

Palantir를 아시나요?  
CIA가 사용하는 데이터 분석 플랫폼입니다.  
우리는 같은 아키텍처를 사용합니다.

**데이터 흐름을 보시겠습니다:**

센서에서 위치, 속도 같은 Raw 데이터가 들어옵니다.  
이것을 온톨로지로 변환하면 의미(Meaning)가 생깁니다.  
'지게차 F-07이 교차로에 접근 중'이라는 의미죠.

SWRL 추론 규칙으로 위험을 판단(Reason)합니다.  
화면의 규칙을 보시면:  
'만약 지게차 2대가 교차로에 접근하고, 속도가 8km/h 이상이면,  
충돌 위험 레벨 HIGH, 음성 명령 발령'

SHACL 정책으로 행동을 결정(Policy)하고,  
실제로 스피커로 음성을 송출(Action)합니다.

**이 모든 과정이 0.2초 안에 일어납니다.**

그리고 가장 중요한 것 - 모든 판단 근거가 기록됩니다.  
왜 이 명령을 내렸는지, 법정에서도 설명할 수 있습니다."

## 슬라이드 7: Human-in-the-loop

### 화면
```
이것은 자율 제어가 아닙니다!

[표]
구분 | 시스템 | 운전자
-----|--------|-------
감지 | ✅ AI | -
판단 | ✅ 규칙 | -
조언 | ✅ 음성 | -
결정 | - | ✅ 최종
조작 | - | ✅ 브레이크
책임 | - | ✅ 운전자

핵심 원칙:
"AI가 조언하고, 사람이 결정합니다"
```

### 발표 (1분)
"여기서 매우 중요한 점을 강조하겠습니다.

**이것은 자율 제어가 아닙니다!**

표를 보시면 역할이 명확히 구분되어 있습니다.

시스템은 위험을 감지하고, 규칙에 따라 판단하고, 음성으로 조언합니다.  
하지만 최종 결정은? **운전자**가 내립니다.  
브레이크를 밟는 것도? **운전자**입니다.  
책임도? **운전자**에게 있습니다.

이것을 우리는 'Human-in-the-loop' 설계라고 부릅니다.

**왜 이것이 중요할까요?**

첫째, 법적 책임이 명확합니다. 사고가 나도 누구 책임인지 분명합니다.  
둘째, 규제가 간단합니다. 자율 주행 관련 복잡한 인허가가 필요 없습니다.  
셋째, 운전자가 수용합니다. '나를 돕는 시스템'이지 '나를 대체하는 시스템'이 아닙니다.

**이것이 우리 시스템의 철학입니다.**  
AI가 조언하고, 사람이 결정합니다."

## 슬라이드 8: ROI 분석

### 화면
```
투자 대비 효과 (연간)

💰 비용 절감:
- 사고 방지: 3억원 (10건 × 3천만원)
- 보험료 절감: 5천만원
- 생산성 향상: 2억원
- 법적 비용: 5천만원
────────────────
총 절감: 5-6억원/년

💵 투자 비용:
- 하드웨어: 1.5억원
- 소프트웨어: 5천만원
- 설치/교육: 3천만원
────────────────
총 투자: 2.3억원

📊 ROI: 117%
⏱️ 회수 기간: 6개월
```

### 발표 (1분 30초)
"이제 가장 중요한 이야기를 하겠습니다. 돈 이야기입니다.

**먼저 절감 효과를 보겠습니다:**

사고 방지만으로 연간 3억원이 절감됩니다.  
현재 연간 12건 발생하는 사고를 2건으로 줄일 수 있습니다.  
10건 방지, 건당 3천만원, 총 3억원입니다.

보험료도 30-40% 줄어듭니다. 5천만원 절감.

운행 효율이 15% 향상되어 생산성이 올라갑니다. 가치로 환산하면 2억원.

법적 비용, 소송, 보상금도 줄어듭니다. 5천만원에서 1억원.

모두 합치면 **연간 5억에서 6억원이 절감됩니다.**

**투자 비용은 얼마일까요?**

하드웨어 1.5억원입니다. UWB 센서, LiDAR, CCTV 업그레이드.  
소프트웨어 개발 5천만원.  
설치와 교육 3천만원.

총 2.3억원입니다.

**계산해볼까요?**

연간 절감 5억원, 초기 투자 2.3억원.  
ROI는 117%.  
회수 기간은? **단 6개월입니다.**

자율 주행 지게차 1대 값으로,  
전체 물류센터의 안전을 확보할 수 있습니다.

**이것이 우리 솔루션의 경제성입니다.**"

## 슬라이드 9: 구현 로드맵

### 화면
```
3단계 구현 계획

Phase 0: POC (2-3주)
- 교차로 1곳, 지게차 2대
- 효과 검증
- 위험 없는 시작 ✅

Phase 1: MVP (2개월)
- 전체 시스템 구축
- 센서 설치
- 온톨로지 개발

Phase 2: Full Deployment (1-2개월)
- Advisory → Directive → Enforcement
- 단계별 배포
- 운영 전환

총 기간: 3-4개월
```

### 발표 (1분)
"그렇다면 언제 시작할 수 있을까요?

**우리는 3단계 접근 방식을 제안합니다:**

**Phase 0는 POC입니다.**

단 2-3주면 효과를 검증할 수 있습니다.  
교차로 1곳, 지게차 2대만으로 시작합니다.  
비용은 최소화하고, 위험도 없습니다.

이 단계에서 시스템이 실제로 작동하는지, 효과가 있는지 확인합니다.  
만약 만족스럽지 않다면? 여기서 멈출 수 있습니다.

**Phase 1은 전체 시스템 구축입니다.**

2개월이 걸립니다.  
센서를 설치하고, 온톨로지를 개발하고, 대시보드를 만듭니다.

**Phase 2는 운영 전환입니다.**

처음에는 Advisory 모드로 시작합니다. 권고만 제공하죠.  
사용자가 익숙해지면 Directive 모드로 올립니다. 지시를 내립니다.  
마지막으로 가능한 장비에는 Enforcement까지 활성화합니다.

**총 기간은 3-4개월입니다.**

내년 이맘때쯤이면 이미 사고를 방지하고 있을 것입니다."

## 슬라이드 10: 확장 가능성

### 화면
```
다른 산업으로 확장

✅ 공장 (지게차 + AGV)
✅ 항만 (크레인 + 차량)
✅ 공항 (지상 장비)
✅ 건설 현장 (중장비)
✅ 주차장 (차량 + 보행자)

같은 기술, 다른 응용:
- 온톨로지만 수정
- 센서는 동일
- 규칙 추가 용이

[각 산업 이미지 5장]
```
