OntologyEngine — 5단계 계층 추론 (Observation→Intent)
Observation → Event (9종) → Situation → Risk Score (0–100) → Intent 흐름. 열폭주, 리튬 석출, 내부 단락, SEI 성장, 과방전, 급속충전 스트레스 감지. 위험까지 남은 시간 예측(dT/dt 기반).

위험도 스코어 기반 4단계 대응 (OK/Warning/Danger/Critical). 충전 전류 자동 감소 (25%→40%→60%→차단), 냉각 팬/Peltier ON, 셀 밸런싱, MOSFET 긴급 차단. CAN 프레임 자동 생성.

Python · Layer 3 OntologyEngine.infer() 
def infer(self, raw: SensorReading, sig: ProcessedSignal) -> OntologyInference:
    """5단계 Operational Ontology 추론"""

    # Level 1: Observation
    obs = f"셀온도 {raw.temp_max:.1f}°C · ΔV {sig.cell_imbalance:.1f}mV · "
          f"임피던스 {raw.impedance_mohm:.1f}mΩ · 전류 {raw.current_a:.1f}A"

    # Level 2: Events (9가지 이벤트 감지)
    events = []
    if sig.kalman_temp > th['temp_warn']:   events.append(ThermalRiseEvent)
    if sig.kalman_temp > th['temp_danger']: events.append(ThermalRunawayEvent)
    if sig.cell_imbalance > th['delta_v_warn']: events.append(VoltageImbalanceEvent)
    if imp_delta > th['imp_danger_delta'] and sig.dt_dt > th['dtemp_warn']:
        events.append(LithiumPlatingEvent)

    # Level 4: Risk Score (0–100)
    risk = 8.0
    if   t > th['temp_critical']: risk += 50
    elif t > th['temp_danger']:  risk += 35
    elif t > th['temp_warn']:    risk += 18
    if dt > th['dtemp_danger']:    risk += 20
    if has_short:  risk *= 1.5   # 내부 단락 가중치
    if has_plating and has_thermal: risk *= 1.3

    # Level 5: Intent → MCP 명령 결정
    if   rl == CRITICAL: intent = "긴급 차단 + 제조사 알림"
    elif rl == DANGER:   intent = "충전 전류 감소 + 냉각 팬 ON + 알림"
    else:                intent = "정상 운용 계속 — 모니터링"

def decide(self, infer: OntologyInference, raw: SensorReading) -> MCPCommand:
    """위험도 기반 MCP 제어 명령 결정"""
    rl    = infer.risk_level
    score = infer.risk_score

    if rl == RiskLevel.WARNING:            # 스코어 25~59
        reduced = max(0.5, curr * 0.75)
        cmd.charge_current_a = reduced
        cmd.fan_pct          = 30           # 30% 예비 ON
        cmd.alert_level      = "warn"

    elif rl == RiskLevel.DANGER:           # 스코어 60~79
        reduced = max(0.3, curr * 0.40)
        cmd.charge_current_a = reduced
        cmd.fan_pct          = 100          # 100% 냉각
        cmd.cell_balance     = True
        cmd.alert_level      = "critical"

    else:  # CRITICAL ≥ 80
        cmd.charge_current_a = 0.0          # 완전 차단
        cmd.mosfet_cut       = True         # MOSFET 긴급 차단!
        cmd.fan_pct          = 100

# CAN 프레임 자동 생성 (충전 전류 명령 예시)
cur_raw = int(cmd.charge_current_a * 10)
frame = {
    'id':   0x18FF50E5,
    'data': [0x01, (cur_raw >> 8) & 0xFF, cur_raw & 0xFF,
              0x00, 0xFF, 0xFF, 0xFF, 0xFF]
}
# → CAN TX: 0x18FF50E5 [8] 01 0C 00 00 FF FF FF FF  (1.2A)
Python · Main Loop BEIAgent._tick_once() — 100ms 자율 루프 
async def _tick_once(self):
    # ── Layer 1: 센서 수집 (CAN / SMBus) ─────────
    raw  = self.hw.read_sensors()

    # ── Layer 2: Kalman 필터링 ────────────────────
    sig  = self.signal.process(raw)

    # ── Layer 3: Ontology 5단계 추론 ─────────────
    infer = self.ontology.infer(raw, sig)

    # ── Layer 4: Agentic MCP 명령 결정 ───────────
    cmd  = self.agent.decide(infer, raw)

    # ── Layer 5: CAN 명령 전송 ───────────────────
    if cmd.action != MCPAction.NONE:
        for frame in cmd.can_frames:
            self.hw.send_can_command(frame['id'], frame['data'])
            await asyncio.sleep(0.002)   # 2ms 간격

    # ── 위험 시 클라우드 MQTT/HTTP 알림 ──────────
    if infer.risk_level in (DANGER, CRITICAL):
        self.cloud.send_alert(infer, raw, cmd)
Quick Start
3단계 빠른 시작 가이드
📦 의존성 설치 (Raspberry Pi CM4 / Jetson)
# Python 패키지 설치
$ pip install python-can smbus2 paho-mqtt requests numpy

# Raspberry Pi: CAN 인터페이스 활성화
$ sudo ip link set can0 up type can bitrate 500000
$ sudo ifconfig can0 up

# Jetson: CAN 활성화 (Jetson AGX Orin 기준)
$ sudo modprobe can && sudo modprobe can_raw && sudo modprobe mttcan
$ sudo ip link set can0 up type can bitrate 500000 dbitrate 2000000 berr-reporting on fd on
1
시뮬레이션 모드로 테스트
실제 하드웨어 없이 bei_agent.py 동작 확인.
python3 bei_agent.py --simulate --scenario thermal
열폭주 시나리오를 실행하면 MCP 명령이 자동으로 생성되고 로그에 CAN TX 프레임이 출력됩니다.
2
실제 BMS 연결 (CAN 인터페이스)
드론/로봇/EV 배터리 팩의 CAN 포트를 BEI 보드에 연결.
sudo python3 bei_agent.py --interface can0 --baudrate 500000
자동으로 CAN 프레임 수신 시작, 100ms 주기로 Ontology 추론 실행.
3
클라우드 연동 + 서비스 등록
제조사 서버 URL과 디바이스 ID를 설정하면 위험 발생 시 자동 알림 전송.
sudo python3 bei_agent.py --interface can0 --device-id DRONE-001 --cloud-url https://yourserver.com/api/alert
systemd 서비스 등록으로 부팅 시 자동 실행 가능: sudo systemctl enable bei-agent && sudo systemctl start bei-agent
